## [Unreleased]

### Added

- Overdue + Web Push sprint (ADR-0071 / ADR-0072 / ADR-0073 / ADR-0074
  / ADR-0075). `Serving` is removed from the `Ticket` discriminated
  union — `Called` transitions directly to `Served` or, on the alarm
  timer, to a new `Overdue` state where a bounded nudge loop fires up
  to `MAX_NUDGES` times before flipping to `NoShow`. Reservation-lane
  tickets still in `Waiting` past `appointmentAt + grace` are
  auto-cancelled with `reason === "appointment_lapsed"`. Customer
  notifications gain Web Push (VAPID, RFC 8291 aes128gcm) via a new
  `@booking/push` workspace package; subscriptions are ticket-scoped
  and payload-anonymous (`{ v, kind, displaySeq }`), reaped on
  terminal transition and on the push service's 404/410. `/staff`
  Kanban gains an Overdue column; `calledAlert` dedups on
  `(calledAt, nudgeCount)` so Nudged events re-fire the chime.
  QueueShop's `alarm()` runs as a 4-tick sweep (Called→Overdue,
  Nudge, Overdue→NoShow, appointment-lapse) with strict
  inequality cutoffs to avoid Tick 2/3 firing in the same alarm.
- Slot-booking time-axis sprint (ADR-0066 / ADR-0067 / ADR-0068).
  Reservation lane gains `appointmentAt: NullOr<Instant>`; EDF
  promotes the eligible reservation head past the static
  `priority > walkIn > reservation` chain when within `grace`
  (default 5min, env `EDF_GRACE_MINUTES`). Customer-side check-in
  audit (`POST /api/v1/tickets/:id/check-in`), `/issue` page gains
  a folded reservation expand with date tabs + slot grid, `/ticket`
  countdown + 「到着しました」 button, `/staff` Kanban slot-time
  chip per reservation card. Door-QR walk-in entry: print the
  `/issue` URL as a 2-D barcode and display at the entrance — no
  in-store kiosk required, no env var, no code change.
- Phase 3 / PR#7 — GraphQL functor migration. Replaced the Pothos
  builder with a Schema-to-GraphQL functor (`derive/graphql.ts`)
  that lifts every wire type from the existing `Effect.Schema`
  declarations. The printed SDL is pinned by a byte-equal test
  against `apps/default/schema.graphql`. ADR-0041 is the design
  write-up; the full conclusion is appended as a Phase 3 PR#8
  retrospective on the same ADR.
- Phase 3 / PR#8 — DX hardening + observability completion. Adds:
  - `RuntimeMode` port (ADR-0042) — env-indexed Layer dispatcher
    (`"dev"` / `"prod"`) selecting verbose vs. terse adapters.
  - `ErrorRedaction` port (ADR-0043) — `extensions.cause` redactor
    for the GraphQL error response (dev-mode preview / prod-mode
    terminal).
  - `LogSampler` port — severity-keyed access-log sampler.
  - Telemetry semconv attribute table on every layer of the trace
    tree: `graphql.operation.*` on the resolver span,
    `messaging.cloudflare.do.dispatch` (new span) with
    `messaging.system="cloudflare.do"` /
    `rpc.method=<envelope.tag>` on the DO RPC client,
    `usecase.invocation.kind` on the use-case span, `db.system.name=
    "d1"` on the audit-write span. ADR-0038 retrospective addendum
    documents the table.
  - Miniflare in-process integration suite under
    `apps/default/test/integration/` covering DO crash recovery and
    `holdSlot` end-to-end through the cross-realm sanitiser.
  - Justfile inner-loop recipes: `dev-up`, `smoke-all`,
    `trigger-scheduled`, `test-integration`, `gen-error-docs`,
    `log-tail`.
  - `apps/web` real test surface: GraphQL client wrapper test,
    gql.tada typegen pin, paraglide locale parity + error-tag
    coverage.
- Top-level docs: `docs/observability.md`, `docs/dev-workflow.md`,
  refreshed `CONTRIBUTING.md`.

### Fixed

- Phase 3 / PR#8 — DO RPC envelope sanitiser (ADR-0044). Fixes
  `DataCloneError: Could not serialize object of type "Object"`
  observed on `effect@4.0.0-beta.62` against `wrangler dev --local`.
  The fix is a targeted shallow boundary transform that normalises
  the two structured-clone-incompatible fields (BigInt request id
  → sigil string; cross-realm / null-prototype objects → same-realm
  plain `{}`) without erasing Effect's class-instance brands.
